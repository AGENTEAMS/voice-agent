חמישה שומרים שמחזיקים את המערכת

  • הלוגיקה ב-SQL, לא בפרומפט - הכלים קוראים לפונקציות Postgres; הסוכן רק קורא להן, לא מחשב.
  • חוזה כלים - פעולה רק דרך קריאה אמיתית לכלי; אסור לנקוב בשעה בלי לבדוק זמינות.
  • שער אישור - שינוי רק אחרי "כן" ברור; ההחלטה האחרונה היא הקובעת.
  • שומר דטרמיניסטי (n8n) - כל שיחה שהסתיימה משאירה שינוי-סטטוס או שיחה חוזרת; אחרת ← חיוג חוזר, או סימון לטיפול אנושי.
  • תיעוד מלא - כל החלטה נכתבת ל-Postgres. אף אורח לא נופל בין הכיסאות.
